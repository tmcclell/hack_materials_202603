🤖 Lab 3: Let the Agent Drive

Experience Agent Mode building a multi-file feature from plain English.

⏱ 25 minutes 🟡 Intermediate
Goal: Experience Agent Mode building a multi-file feature from a single plain-English prompt. Watch Copilot plan, create files, run commands, and self-correct.
1

Enable Agent Mode

  • Open Copilot Chat
  • Look for the mode selector at the top of the chat input — you'll see options like Ask, Edit, and Agent
  • Switch to "Agent" mode
⚠️ Key difference: Unlike Ask or Edit mode, Agent mode can create files, run terminal commands, and iterate on its own work. It will ask for your confirmation before executing commands.
The mode selector shows "Agent" and the chat input may show a slightly different UI indicating agent capabilities.
If you don't see the Agent option, check that you're on the latest Copilot Chat extension. The mode selector is usually a dropdown or toggle near the chat input area.
2

Give the Agent a Task

  • Type the following prompt into the Agent chat:
Create a TodoList module with:
- A TodoItem interface (id, title, completed, createdAt)
- A TodoService with CRUD operations using an in-memory array
- A basic Express router at /api/todos
- Include input validation
  • Press Enter and watch the agent work
  • It will start creating files, writing code, and may run commands
  • Approve any terminal command prompts when asked
Multiple files created — likely a model/interface file, a service file, and a routes file. The agent may also create or modify package.json and install Express.
If the agent seems stuck or asks a clarifying question, answer it and let it continue. If it generates everything in a single file, try: "Can you split this into separate files — one for the interface, one for the service, and one for the routes?"
3

Watch the Decision-Making

As the agent works, observe its process:

  • 🧠 Planning: Does it describe what files it will create?
  • 📁 Order: Does it create files in a logical sequence (model → service → routes)?
  • 🔧 Tooling: Does it run the TypeScript compiler or linter to check for errors?
  • 🔄 Self-correction: If it finds issues, does it fix them automatically?
The agent shows a clear reasoning process — stating what it plans to do, then executing step by step. You may see it fix import paths or type errors it catches.
If you're not seeing the reasoning steps, look for a "thinking" or "plan" section in the agent's output. Some models show more verbose reasoning than others — try switching to a different model if you want more detail.
4

Test the Agent's Work

  • In the same Agent conversation, type:
Now write tests for the TodoService
  • Watch the agent:
    • Create a test file
    • Write test cases for each CRUD operation
    • Potentially run the tests
    • Fix any failures it discovers
A test file with tests for creating, reading, updating, and deleting todos. If the agent runs the tests, they should pass (it will fix issues if they don't).
If the agent doesn't run the tests automatically, ask it: "Run the tests and fix any failures." If there are dependency issues, ask: "Install any missing test dependencies and try again."
5

Review and Reflect

  • Read through all the generated code
  • Ask yourself:
  • ✅ Is the code style consistent across files?
  • ✅ Did it follow any copilot-instructions.md you may have set up?
  • ✅ Are the imports correct and organized?
  • ✅ Is the input validation thorough?
  • ❓ What would you change or improve?
You should have a working multi-file feature with clear separation of concerns, reasonable validation, and passing tests.

🚀 Finished Early?

Test a cross-cutting change! In the same Agent conversation, type:

Add a priority field (high/medium/low) to TodoItem and update all related code

Watch how the agent handles updating the interface, service logic, route validation, and tests — all from a single prompt. This is where Agent Mode really shines.
