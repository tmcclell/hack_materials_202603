If You Have a Use Case

Primary track — recommended if you have a real project to work on.

  1. Bring your real project/repo — something you're actively working on
  2. Identify one feature or improvement to tackle in the time available
  3. Use Agent Mode, Plan Mode, or Autopilot to accelerate your work
  4. Prepare a 5-minute demo for Show & Tell

💡 Tip: Scope small! It's better to have one complete feature than three half-done ones. Pick the thing that will have the most impact and demo the best.

🎯

Fallback Track: No Use Case? No Problem!

Pick one of these three guided challenges. Each one is scoped to be completable in 3 hours with Copilot assistance.

Challenge A: REST API from a PRD

"Build a bookmark manager API with folders, tags, and search. Users can save URLs with titles and descriptions, organize into folders, tag for filtering, and full-text search."
  • Use Plan Mode to break the spec into tasks
  • Use Autopilot to implement each task
  • Suggested stack: Express + TypeScript + in-memory store (or SQLite)

Challenge B: Repo Health Automation

"Build a CLI tool that audits a GitHub repo: checks for stale branches (>30 days), missing README sections, test coverage gaps, and outdated dependencies. Output a markdown report."
  • Use Agent Mode for the CLI scaffold
  • Use Chat for individual health checks
  • Suggested stack: Node.js CLI with Commander + Octokit

Challenge C: Internal Tool Generator

"Build a team standup bot — a simple web app where team members post daily updates (what they did, what they'll do, blockers). Shows a dashboard of today's updates."
  • Use Agent Mode end-to-end
  • Use /tests for test coverage
  • Suggested stack: Express + HTML/CSS frontend (or React if ambitious)
🎤

Show & Tell Format

4:15 – 4:50 PM · ~5 minutes per team

Demo Template:
  1. What was the challenge? — Brief description of what you set out to build
  2. What Copilot features did you use? — Agent Mode? Prompt files? Custom instructions? Which were most helpful?
  3. Live demo — Show what you built, working or not!
  4. What surprised you? — Best moment? Worst moment? What did you learn?

📸 Remember: Demos don't have to be polished. Sharing what you learned is just as valuable as sharing what you built.

💡

Tips for a Great Hack

🗺️ Start with Plan Mode Let Copilot help you break down the problem before writing code. A good plan saves hours.
🤖 Delegate the boring parts Let Copilot write boilerplate, tests, and docs. Focus your brain on architecture and design decisions.
🔍 Trust but verify Review everything Copilot generates. It's fast but not infallible — catching one bug early saves a lot of time.
🆘 Ask for help If you're stuck for more than 10 minutes, flag a facilitator. That's what they're here for!
💾 Commit early, commit often Use git to save your progress. Nothing worse than losing work to a stray Ctrl+Z.
🎯 Scope ruthlessly It's better to ship one complete feature than to have three features at 50%. Pick the most demoable path.
