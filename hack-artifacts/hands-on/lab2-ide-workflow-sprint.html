⚡ Lab 2: IDE Workflow Sprint

Master prompt engineering, refactoring, test generation, and documentation.

⏱ 20 minutes 🟡 Intermediate
Goal: Master prompt engineering, refactoring, test generation, and documentation — the core Copilot workflows you'll use every day.
1

Comment-Driven Prompting

  • Create a new file called calculator.ts
  • Type the following detailed comment:
// Calculator class with add, subtract, multiply, divide methods.
// Division by zero should throw an error.
// All methods should accept two numbers and return a number.
  • Press Enter and let Copilot generate the class from the comment
  • Accept suggestions with Tab, keep pressing Enter for more
A full Calculator class with add, subtract, multiply, and divide methods. The divide method includes a check for division by zero.
If Copilot only gives you one method at a time, accept it and press Enter — it should suggest the next one. You can also write class Calculator { first to give it more structure to work with.
2

Refactor with Chat

  • Select the entire Calculator class
  • In Chat, type:
Refactor this to use arrow functions and add input validation for non-numeric values
  • Review the diff that Copilot shows
  • Click Accept (or Apply) to apply the changes
Cleaner code with arrow function syntax and input validation that checks for non-numeric values before performing operations.
If you don't see a diff view, try using Edit mode (the pencil icon in Chat) instead of Ask mode. You can also copy the refactored code and paste it manually.
3

Generate Tests with /tests

  • Select the Calculator class
  • In Chat, type:
/tests
  • Review the generated test suite
  • Look for edge cases: division by zero, negative numbers, decimals
A Vitest (or Jest) test suite covering normal operations, edge cases (division by zero), and potentially invalid input scenarios.
If /tests isn't recognized, try typing: "Write unit tests for this code using Vitest". Make sure the Calculator class code is selected before invoking.
4

Generate Docs with /doc

  • Select the Calculator class
  • In Chat, type:
/doc
  • Review the generated documentation
JSDoc comments on every method — including @param types, @returns descriptions, and @throws for the division-by-zero case.
If /doc isn't recognized, try: "Add JSDoc documentation to every method in this class". Place your cursor inside the class before the first method and type /** — Copilot may auto-complete the docs.
5

Prompt Engineering: #file and #selection

  • Create a new file called utils.ts
  • In Chat, type the following prompt (referencing your existing file):
Using the same coding style as #file:calculator.ts, create a StringUtils class
with capitalize, reverse, and isPalindrome methods
  • Review: Does it match the coding style of your Calculator class?
A StringUtils class that mirrors the style of your Calculator — same patterns for arrow functions (if refactored), validation, and JSDoc comments.
If #file: doesn't auto-complete, make sure the file calculator.ts is in your workspace. Try typing #file: and wait for the dropdown to appear. You can also try #file:./calculator.ts.
