🚀 Lab 1: Quick Start

Verify Copilot is active and experience the three core interaction modes.

⏱ 10 minutes 🟢 Beginner
Goal: Verify Copilot is active and experience the three core interaction modes — inline completions, Chat, and the model picker.
1

Verify Copilot is Active

  • Open VS Code
  • Check for the Copilot icon in the status bar (bottom-right)
  • Open the Copilot Chat panel: press Ctrl+Alt+I or click the chat icon
  • Type the following into Chat:
Hello, are you working?
  • Verify you get a conversational response
Copilot responds conversationally, confirming it's active and ready to help.
If you don't see the Copilot icon, make sure the GitHub Copilot extension is installed and you're signed in. Try Ctrl+Shift+P → "GitHub Copilot: Sign In".
2

Inline Completions

  • Create a new file called hello.ts
  • Type the following comment on the first line:
// Function that greets a user by name and returns the greeting
  • Press Enter and start typing:
function greet
  • Wait for ghost text to appear (dimmed suggestion)
  • Press Tab to accept the completion
Copilot completes the function — something like function greetUser(name: string): string { return `Hello, ${name}!`; }
No ghost text? Make sure the file is saved as .ts (TypeScript). Try typing a bit more of the function signature to trigger the suggestion. You can also press Ctrl+Space to manually trigger suggestions.
3

Chat: Explain Code

  • Select the entire function you just created
  • Open Chat (if not already open)
  • Type:
/explain
  • Read the explanation Copilot provides
Copilot explains what the function does — its purpose, parameters, return value, and any logic inside.
Make sure you have the code selected (highlighted) before running /explain. If Chat says it doesn't understand, try selecting the code and using "Explain This" from the right-click context menu instead.
4

Check Your Model

  • Look at the model picker in the Chat panel (usually at the top or bottom of the chat input)
  • Note which model is currently selected (likely "Auto")
  • Click it and try switching to a different model — notice the dropdown options
  • Switch back to Auto (recommended for the 10% discount!)
You can see and switch between available models. "Auto" is the recommended default.
If you don't see the model picker, make sure you're using the latest version of the GitHub Copilot Chat extension. Update via Ctrl+Shift+X → search "GitHub Copilot Chat" → Update.

🚀 Finished Early?

Try workspace-level context! In Chat, type:

@workspace What files are in this project?

This shows Copilot can see your entire project structure, not just the open file. It's a great way to orient yourself (and Copilot) in a new codebase.
