⚙️ Lab 4: Configure & Extend

Create Copilot Instructions and prompt files that customize Copilot's behavior.

⏱ 15 minutes 🔴 Advanced
Goal: Create Copilot Instructions and reusable prompt files that customize Copilot's behavior to match your team's coding standards and workflows.
1

Create copilot-instructions.md

  • In your project root, create a folder: .github/
  • Inside that folder, create a file: .github/copilot-instructions.md
  • Add the following content:
# Copilot Instructions

## Coding Standards
- Always use TypeScript strict mode
- Use JSDoc comments on all public functions
- Use const by default, let only when needed
- All API responses: { success: boolean, data?, error? }
- Error messages must be user-friendly (no stack traces)
- Use descriptive variable names (no single letters except in loops)
  • Save the file
File created at .github/copilot-instructions.md. Copilot will automatically read this file and apply these standards to all suggestions in this workspace.
Make sure the file is in the .github folder at the root of your workspace (not inside src/ or another subfolder). The file must be named exactly copilot-instructions.md — Copilot looks for this specific name.
2

Test the Instructions

  • Open Copilot Chat in Agent mode
  • Type:
Create a UserService with getUser and createUser methods
  • When the code is generated, check:
  • ✅ Does it use JSDoc comments on the methods?
  • ✅ Does it use const by default?
  • ✅ Does the API response match { success: boolean, data?, error? }?
  • ✅ Are error messages user-friendly?
  • ✅ Are variable names descriptive?
The generated code follows your defined standards — JSDoc comments, const usage, standardized response format, user-friendly errors, and descriptive names.
If the instructions aren't being followed, try reopening the Chat panel or reloading VS Code (Ctrl+Shift+P → "Developer: Reload Window"). You can also explicitly reference the file: "Follow the instructions in #file:.github/copilot-instructions.md".
3

Create a Prompt File

  • Create a subfolder: .github/prompts/
  • Create a file: .github/prompts/add-api-endpoint.md
  • Add the following content:
# Add API Endpoint

Create a new REST API endpoint with:
1. Input validation using a validate function
2. Service layer for business logic
3. Express route handler
4. Vitest tests with at least 3 test cases
5. JSDoc documentation

Follow the patterns in the existing codebase.
  • Save the file
File created at .github/prompts/add-api-endpoint.md. This is a reusable template you can reference any time you need a new endpoint.
The .github/prompts/ folder is the conventional location for prompt files. Make sure the folder exists before creating the file. You can create both in the terminal: mkdir -p .github/prompts.
4

Use the Prompt File

  • In Chat, reference the prompt file you just created:
Using #file:.github/prompts/add-api-endpoint.md, add a /api/comments endpoint
  • Watch Copilot follow the template steps:
  • 1️⃣ Input validation function
  • 2️⃣ Service layer with business logic
  • 3️⃣ Express route handler
  • 4️⃣ Vitest tests (at least 3)
  • 5️⃣ JSDoc documentation
A complete /api/comments endpoint with all five components from the template — validation, service, routes, tests, and documentation.
If #file: doesn't resolve, try the full relative path: #file:.github/prompts/add-api-endpoint.md. You can also copy-paste the prompt file content directly into Chat as a workaround.

🚀 Finished Early?

Create a second prompt file! Make .github/prompts/refactor-to-pattern.md:

# Refactor to Strategy Pattern

Refactor the selected code to use the Strategy pattern:
1. Extract the varying behavior into a Strategy interface
2. Create concrete strategy implementations
3. Modify the context class to accept a strategy
4. Update all call sites
5. Add tests for each strategy

Maintain backward compatibility.

Then test it: select some code with conditional logic and ask Chat to apply the prompt.
