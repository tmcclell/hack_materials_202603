Model Governance & Policy + Audit & Compliance

⏱ 10 minutes total
🎯 Key Message
"You control which models your developers use, and how much they cost."

Model Governance & Policy

5 minutes
  • Model selection is controlled at the org/enterprise level
  • Admins can restrict models via policy (block specific models or allow only approved ones)
  • Content exclusions: prevent Copilot from accessing sensitive files/repos
  • IP indemnity: only applies to certain models and plans
  • Audit logging: every Copilot interaction can be tracked
🎯 Key Message
"AI-generated code needs the same review rigor as human code — maybe more."

Audit & Compliance

5 minutes
  • Session trails: Copilot logs show what was generated and when
  • Code review hygiene: AI-generated PRs need human review
  • Building trust incrementally: start with Agent Mode (interactive), graduate to Autopilot
  • License scanning: Copilot's public code filter
  • Compliance considerations for regulated industries

❓ Anticipated Questions

"Can we block developers from using expensive models?"
Yes, admin policies can restrict model access
"Is our code sent to external servers?"
Copilot processes in GitHub's infrastructure, no code is stored for training on Business/Enterprise plans
"Can we see who's using the most premium requests?"
Yes, usage reports show per-user consumption
