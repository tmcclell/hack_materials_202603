Skills & MCP

Extending Copilot with custom tools, data sources, and the Model Context Protocol

1 What Are Skills?

🔌 Plugins for Copilot

Skills extend Copilot's capabilities beyond code generation. Think of them as plugins that give Copilot new abilities — like querying a database, calling an API, or accessing internal tools.

🧩 How They Work

When you use a skill, Copilot sends your request to a specialized tool that processes it and returns structured results. The AI then uses those results to form its response to you.

📦 Built-in Examples

  • @workspace — search your codebase
  • @terminal — interact with the terminal
  • @vscode — VS Code settings & actions
  • #file — reference specific files
  • #selection — reference selected code
💡 Key insight: Skills turn Copilot from a code suggestion tool into an intelligent assistant that can interact with your entire development ecosystem.

2 Model Context Protocol (MCP)

📡 An Open Standard for AI Tool Integration

MCP is an open protocol that standardizes how AI assistants connect to external tools and data sources. Instead of building custom integrations for every tool, MCP provides a universal interface.

Architecture Overview

🤖 Copilot

VS Code / CLI / Chat

←→ MCP Protocol
🔗 MCP Client

Built into Copilot

←→ JSON-RPC / stdio
⚙️ MCP Server

Local or remote process

→
📊 External Tools

APIs, DBs, services

🔧 How MCP Works

  • MCP servers expose tools (functions the AI can call)
  • MCP servers expose resources (data the AI can read)
  • Communication via JSON-RPC over stdio or HTTP
  • Servers run locally or remotely
  • Configuration in .vscode/mcp.json or user settings

📝 Config Example

// .vscode/mcp.json { "servers": { "github": { "command": "github-mcp-server", "args": ["stdio"] }, "postgres": { "command": "mcp-server-postgres", "args": ["$DB_URL"] } } }

3 Where This Is Heading

🏗️ Custom MCP Servers for Your Org

Build MCP servers tailored to your organization's internal tools — CI/CD systems, deployment pipelines, monitoring dashboards, wiki systems. Copilot becomes fluent in your infrastructure.

🔗 Internal API Access Through Copilot

Imagine asking Copilot "What's the status of the production deployment?" or "Show me the error rates from the last hour" — and getting real answers from your internal APIs.

🌍 A Universal Tool Ecosystem

As MCP adoption grows, an ecosystem of shared servers emerges — install a server, and Copilot instantly gains new capabilities. Like npm for AI tools.

📋 Note for this session: This is presentation-only — the customer is exploring MCP concepts but is not building custom MCP servers yet. The goal is awareness of what's possible.

4 Copilot Extensions

🛒 GitHub Marketplace

Browse and install Copilot extensions from the GitHub Marketplace. These extensions add specialized capabilities directly into your Copilot experience.

  • One-click install from Marketplace
  • Curated and community extensions
  • Use via @extension-name in chat
  • Org admins control which extensions are allowed

🛠️ Custom Extensions (SDK)

Build your own extensions using the Copilot Extensions SDK. Create specialized agents for your team's unique needs.

  • Build with Node.js, Python, or any language
  • Access org-specific data and tools
  • Deploy as GitHub Apps
  • Combine with MCP for maximum power

🔄 MCP vs Extensions

Both extend Copilot, but they serve different roles:

MCP Tool integration — give Copilot access to data & actions
Extensions Experience customization — custom agents, workflows, UIs
