🤖 Autonomous AI & Multi-Agent Systems

Autonomous Copilot & Agent Teams

⏱️ Duration: 25 min  |  Demo 07 — Instructor Script

🎬 Setup

Have the following ready before you begin:

  • VS Code open with the demo-app/ project (Emoji Battle Arena — TypeScript/Express)
  • Terminal visible inside VS Code — project builds and runs cleanly
  • GitHub Copilot Chat panel open in VS Code
  • GitHub.com open in a browser tab — logged in, with the demo repo visible
  • Browser tab with github.com/bradygaster/squad ready

Section 2

🎯 Section 2 — Live Demo: Autopilot

⏱️ 8 min

2

2  ⚡ Start Autopilot Task

💬 Say: "Autopilot takes Agent Mode and removes the training wheels. It makes decisions and executes without asking permission."

Enable Autopilot mode in VS Code (if available) or use Agent Mode without confirmations.

⌨️ Type: 
Add a complete tournament bracket system to the Emoji Battle Arena. Create a Tournament model with bracket, rounds, and results. Create TournamentService with createTournament (accepts fighter IDs, generates bracket), playRound (simulates all matches in a round), and getTournamentResults. Create Express routes at /api/tournaments. Write comprehensive tests. Update the browser UI to show tournaments.

🛑 LET IT RUN — don't touch anything. Narrate what's happening as Copilot works.

Point out: file creation → code generation → test writing → UI updates — all autonomous.

👀 What They See: Full autonomous execution — file creation, code writing, test running, UI updates — all without any human interaction.
💡 Teaching Point: Autopilot = end-to-end execution from plain English. One prompt → complete feature.
3

3  🔍 Review Results

Walk through the files Copilot created — models, services, routes, tests, UI changes.

Run tests in the terminal to show they pass.

Open the browser to show the tournament feature live in the UI.

👀 What They See: A working tournament feature — from zero code to a functional feature in minutes. Tests pass. UI works.
💡 Teaching Point: For well-defined tasks, Autopilot can deliver faster than typing individual prompts. The key is a clear, detailed instruction.
Section 3

☁️ Section 3 — Copilot Coding Agent

⏱️ 5 min

4

4  🌐 Show GitHub Coding Agent

Switch to GitHub.com in the browser.

💬 Say: "What if you don't even need VS Code open? The coding agent works in the cloud."

Show an issue (or create one live): "Add fighter matchmaking based on skill level"

Assign the issue to @copilot (or show how you would assign it).

If previously triggered, show the PR that the coding agent created — real code changes, tests, docs.

👀 What They See: On GitHub.com — an issue assigned to @copilot, then (if previously triggered) a PR created by the coding agent with real code changes.
💡 Teaching Point: Async, cloud-based autonomous coding. Write an issue, assign to Copilot, come back to a PR. No IDE needed.
Section 4

👥 Section 4 — Squad Pattern

⏱️ 5 min

5

5  🏗️ Introduce Squad

💬 Say: "What if you could have a whole TEAM of AI agents? That's Squad."

Open: https://github.com/bradygaster/squad

🔑 Key Points to Mention:

  • Each team member runs in its own context — independent agent processes
  • Parallel execution of different concerns — frontend, backend, tests all at once
  • Shared decisions and coordination between agents
  • Session persistence across runs — agents remember context

Compare on screen: Single agent (one thing at a time) vs Squad team (frontend + backend + tests simultaneously).

👀 What They See: The Squad repo and README showing the multi-agent architecture — multiple agents working in parallel on different parts of a codebase.
💡 Teaching Point: This is where agentic development is heading — from a single AI assistant to AI development teams working in concert.
Wrap-Up

🎁 Wrap-Up — When to Use Each

⏱️ 2 min

6

6  📊 The Autonomy Cheat Sheet

💬 Say: "Let me give you a cheat sheet for which mode to use when."
Mode Best For Control Level Speed
🤝 Agent Mode Interactive, learning, exploring — you're in the driver's seat 🟢 High — you approve each step Moderate
📋 Plan Mode Complex tasks, need oversight — trust but verify 🟡 Medium — review plan, then execute Moderate
🚀 Autopilot Well-defined tasks, trust the output — hands-off execution 🟠 Low — review after completion Fast
☁️ Coding Agent Async work, PR-based workflow — set it and forget it 🔴 Minimal — review the PR Background
👥 Squad Multi-concern projects, team simulation — AI development team 🔴 Minimal — orchestrated autonomy Fastest ⚡
💡 Teaching Point: Start with more control (Agent Mode), increase autonomy as you build trust. Match the autonomy level to the task complexity and your confidence in the outcome.
