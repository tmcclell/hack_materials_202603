🤖 Live Demo: Copilot Chat

Walk through Copilot Chat's core capabilities — from workspace questions to inline edits — using the Emoji Battle Arena project.

🎬 Setup — Before You Begin

  • VS Code open with the Emoji Battle Arena project (../demo-app/)
  • Copilot Chat panel visible (sidebar or secondary panel)
  • src/services/battleService.ts open in the editor
  • Terminal hidden to maximize screen real estate
  • Font size bumped up for audience visibility (Ctrl +)
1
Step 1

🌐 @workspace Question

Ask Copilot a high-level question about the entire project to show workspace-wide context.

💬 Say:

"Let's ask Copilot about the whole project. Instead of reading every file ourselves, we'll use @workspace to let Copilot index everything and answer."

⌨️ Type in Chat:
@workspace How does the battle system work in this project?
👀 What They See:

Copilot returns a detailed explanation referencing battleService.ts, the Fighter interface, damage calculations, the battle loop flow, and how results are returned. It pulls context from multiple files across the workspace.

💡 Teaching Point:

@workspace indexes the entire project for context. It's like giving Copilot a map of your whole codebase — it can reference any file, interface, or function to answer your questions.

2
Step 2

🔍 /explain — Understand Code

Select the simulateBattle function in battleService.ts and ask Copilot to explain it.

💬 Say:

"Now let's have Copilot explain a specific piece of code. I'll select the simulateBattle function and use the /explain command."

⌨️ Do:

Select the entire simulateBattle function in the editor, then type in Chat:

/explain
👀 What They See:

Copilot breaks down the function step by step — the damage formula, the battle loop with turn-based logic, HP tracking, and the return values. It reads and interprets the selected code like a knowledgeable colleague.

💡 Teaching Point:

Copilot reads the selected code and explains it in plain English. Great for onboarding, code reviews, or understanding unfamiliar code written by someone else.

3
Step 3

🐛 /fix — Find Bugs

Stay on battleService.ts and ask Copilot to find potential bugs in the damage calculation.

💬 Say:

"Let's see if Copilot can find bugs for us. There's a subtle issue hiding in the damage calculation — let's see if Copilot catches it."

⌨️ Type in Chat:
/fix Look for potential bugs in the damage calculation
👀 What They See:

Copilot identifies the operator precedence bug! The expression attacker.attack - defender.defense / 2 should be attacker.attack - (defender.defense / 2). Division binds tighter than subtraction, so without parentheses the defense is halved before subtraction — not the intended behavior.

💡 Teaching Point:

/fix is great for code review — it catches subtle bugs that are easy to miss during manual review. Think of it as a second pair of eyes that never gets tired.

4
Step 4

🧪 /tests — Generate Tests

Open the empty tests/leaderboard.test.ts file and ask Copilot to generate comprehensive tests.

💬 Say:

"What about generating tests? Even for code that doesn't fully exist yet? Let's open our empty leaderboard test file and see what Copilot can do."

⌨️ Do:

Open tests/leaderboard.test.ts (empty file), then type in Chat:

/tests Generate comprehensive tests for the leaderboard service
👀 What They See:

Copilot generates a full test suite with describe/it blocks, proper mocks, and meaningful assertions. Test cases cover adding scores, retrieving rankings, handling ties, and edge cases — even though the service implementation is minimal.

💡 Teaching Point:

Copilot generates test cases even for stub services. It infers expected behavior from naming conventions, interfaces, and project context. Great for TDD or bootstrapping your test suite.

5
Step 5

✏️ Inline Chat — Edit in Place

Use Inline Chat (Ctrl+I) to implement the validateFighterData function directly in the editor.

💬 Say:

"Finally, Inline Chat lets you edit code right where you are. No need to copy-paste from the chat panel — Copilot writes the code directly into your file."

⌨️ Do:

Open src/utils/validators.ts, place cursor inside the validateFighterData function, then press Ctrl+I and type:

Implement this validation function
👀 What They See:

An inline diff appears showing the implementation being added directly in the editor. The audience sees green highlighted lines with validation logic — checking required fields, types, value ranges — appearing right in the function body. Accept or reject with one click.

💡 Teaching Point:

Inline Chat (Ctrl+I) is perfect for quick edits without leaving your code. It's the fastest way to go from "I need this function" to "it's implemented" — right in context, with a diff preview.
