🧠 Prompt Techniques & Chat

🎯 Prompt Engineering + Chat Deep Dive

⏱️ 10 min

🎬 Setup

Have the following ready before starting the demo:

  • VS Code open with the demo-app project (../demo-app/ — TypeScript/Express game API)
  • GitHub Copilot Chat panel open and visible on the side
  • File src/utils/validators.ts open in the editor
  • A fresh Chat thread (clear any previous conversation)
1
Step 1

🎯 Zero-Shot Prompting

📍 In Copilot Chat panel

💬 Say:

"Let's start with the simplest kind of prompt — just ask for what you want."

⌨️ Type:
Create a function that validates an emoji is a single Unicode emoji character
👀 What They See:

Copilot generates a function using regex or Unicode checks — works but may miss edge cases like compound emojis or zero-width joiners.

💡 Teaching Point:

Simple prompt, decent result — but notice it may miss edge cases. We can do better.

2
Step 2

🎯 Few-Shot Prompting

📍 In Copilot Chat panel

💬 Say:

"Now watch what happens when we give examples."

⌨️ Type:
Create a function that validates an emoji.
Here are examples:
  ✅ Valid: 🔥, ⚡, 🛡️
  ❌ Invalid: 'fire', '🔥🔥', '', 'ab'
👀 What They See:

Copilot generates a more robust function that handles the specific cases shown — much better coverage, often with explicit test-like logic for the examples provided.

💡 Teaching Point:

Examples dramatically improve accuracy. A few concrete examples are worth a thousand words of description.

3
Step 3

📝 Comment Anchors

📍 In src/utils/validators.ts — above the sanitizeInput function

💬 Say:

"You don't always need the Chat panel. Detailed comments guide Copilot too."

⌨️ Type:
// Sanitize input: trim whitespace, remove HTML tags
// using regex, escape &, <, >, ", ' characters.
// Return the cleaned string.

Press Enter after the comment and let Copilot autocomplete the function body.

👀 What They See:

Ghost text appears implementing exactly what the comment describes — trim, regex strip tags, character escaping. The implementation follows the comment step-by-step.

💡 Teaching Point:

Detailed comments = better completions. Think of comments as prompts that live in your code and help future developers and Copilot.

4
Step 4

📁 #file References

📍 In Copilot Chat panel

💬 Say:

"What if you want Copilot to follow YOUR code patterns? Use #file references."

⌨️ Type:
Using the patterns from #file:src/services/fighterService.ts, create a PowerUpService with the same structure
👀 What They See:

Generated service matches the existing pattern — same error handling, same response structure, same coding style. It's like having Copilot read your team's playbook.

💡 Teaching Point:

#file gives Copilot specific context from your codebase. It's the difference between generic code and code that fits your project.

5
Step 5

🔄 Multi-turn Conversation

📍 Continue in the same Chat thread

💬 Say:

"Chat remembers context. We can iterate on the same conversation."

⌨️ Type:
Now add a method to apply a power-up to a fighter, modifying their stats temporarily
⌨️ Then Type:
What edge cases should I handle?
👀 What They See:

Copilot builds on the previous PowerUpService, adds the method, then lists edge cases like expired power-ups, stat stacking limits, invalid fighter IDs, and concurrent modifications.

💡 Teaching Point:

Multi-turn conversations build context — each message makes the next one smarter. Think of Chat as a pair programmer who remembers the whole conversation.
