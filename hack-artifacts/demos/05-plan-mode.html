🗺️ Agentic Workflows

📋 Plan Mode

⏱️ Duration: 15 min

🎬 Setup

Have these ready before you begin:

  • VS Code open with the demo-app/ project loaded (Emoji Battle Arena — TypeScript/Express)
  • Copilot Chat panel open and visible
  • Plan Mode toggle visible in the Chat panel (mode selector dropdown)
  • No pending unsaved changes in the editor
  • Terminal clear and ready

📖 Section 1: Concept 5 min

1

Step 1 🤔 What is Plan Mode?

💬 Say: "Plan Mode is 'trust but verify.' The agent proposes steps, you review and approve before anything executes."
  • Show the Plan Mode toggle in VS Code — click the mode selector in the Chat panel
  • Point out the available modes: Ask, Edit, Agent, and Plan
👀 What They See: The Plan Mode toggle/checkbox in the Chat panel — a dropdown or mode selector that switches Copilot's behavior from immediate action to plan-first workflow.
💡 Teaching Point: Explain the autonomy spectrum — there's a range from you doing everything to the agent doing everything. Plan Mode sits right in the sweet spot of supervised autonomy.

🔀 The Autonomy Spectrum

💬 Chat You drive
🤝 Agent Mode You collaborate
📋 Plan Mode You supervise
🚀 Autopilot Full autonomy
← More human control More AI autonomy →
Mode Who Decides? Best For
💬 Chat You ask, AI answers — no code changes Questions, exploration, learning
🤝 Agent Mode AI proposes each action, you approve one-by-one Interactive collaboration, unfamiliar code
📋 Plan Mode AI proposes full plan, you approve upfront, then it executes Complex features, team review, oversight
🚀 Autopilot AI decides and executes — full trust Routine tasks, high-confidence changes

🎯 Section 2: Live Demo 10 min

2

Step 2 🚀 Start a Plan Mode Task

💬 Say: "Let's give it a complex feature and see how it plans. We're going to ask Copilot to add a fighter evolution system — XP, leveling, stat bonuses — across multiple files."
  • Enable Plan Mode in the Chat panel mode selector
  • Type the following prompt into the Chat input:
⌨️ Type: Refactor the Emoji Battle Arena to add a fighter evolution system. Fighters should gain XP from battles and level up, gaining stat bonuses. Add an evolution method to fighterService, update the battle system to award XP, and create tests.
  • Press Enter and wait for the plan to generate
  • Do NOT approve yet — let the audience read the plan
👀 What They See: The agent produces a detailed, numbered plan showing: files to create/modify, what changes go where, and the order of operations. Crucially — nothing has executed yet. No files have changed. It's a proposal, not an action.
💡 Teaching Point: The plan IS the value. Before a single line of code changes, you can review the entire approach, share it with your team, or reject it entirely. This is what makes Plan Mode ideal for complex, multi-file features.
3

Step 3 🔍 Review the Plan

💬 Say: "Look — it's showing us exactly what it will do before doing it. This is perfect for complex features where you want oversight. Let's walk through each step together."
  • Walk through each proposed step with the audience — read them aloud
  • Point out the structure: which files get modified, which get created, the dependency order
  • Now add a constraint — type a follow-up message:
⌨️ Type: Actually, let's cap the max level at 10 and make each level require progressively more XP.
👀 What They See: The plan updates based on the feedback — Copilot revises the relevant steps to incorporate the level cap and progressive XP curve, without losing the rest of the plan. The plan is a living document you can negotiate with.
💡 Teaching Point: You can negotiate with the plan — add constraints, remove steps, reorder things. Think of it like a code review before the code even exists. This is collaborative planning, not just asking for code.
4

Step 4 ✅ Approve and Watch Execution

💬 Say: "I'm happy with this plan. Let's let it execute. Watch how it follows the plan step by step — just like a developer working through a task list."
  • Click the Approve / Execute Plan button
  • Watch the execution proceed step by step
  • Point out each step as it completes — file edits, new file creation, test generation
👀 What They See: Each plan step executes in order with progress indicators. Files open and get edited, new files are created, tests are written and potentially run — all following the exact plan that was approved. The audience can see the plan being realized in real time.
💡 Teaching Point: Plan Mode gives you control without micromanaging — the best of both worlds. You set the direction and constraints upfront, then the agent handles the execution. It's like being an architect who approves blueprints before construction begins.
