🤖 Agent Mode in VS Code

Copilot's most powerful mode — autonomous, multi-step coding

⏱️ 15 min demo 🧩 AI-Assisted Development 🧪 Intro to 25 min lab

🎬 Setup

Have these ready before you start:

  • VS Code open with the demo-app/ project loaded (../demo-app/)
  • GitHub Copilot Chat panel open and visible
  • Agent mode selected in the Chat mode selector
  • File explorer sidebar visible
  • Terminal panel closed (Agent will open it if needed)
  • Clean git state — commit or stash any prior demo changes
1
1

Explain Agent Mode

💬 Say:

"Agent Mode is Copilot's most powerful interactive mode. It can create files, edit code, run terminal commands, and self-correct."

👉 Show the mode selector in VS Code Chat — click through the three modes: Ask → Edit → Agent

👀 What They See:

The Chat panel mode selector with "Agent" highlighted. The mode dropdown shows all three options clearly.

💡 Teaching Point:

Ask = Q&A (answers questions, no file changes) · Edit = targeted changes (edits specific files you point it to) · Agent = autonomous multi-step execution (plans, creates, edits, runs commands, self-corrects)

2
2

Give a Multi-File Task

💬 Say:

"Let's give it a real feature request — something that touches multiple files."

⌨️ Type in Agent Chat: 
Add a power-ups system to the Emoji Battle Arena. Create a PowerUp
interface with id, name, emoji, effect (attack_boost, defense_boost,
heal), and magnitude. Create a PowerUpService with CRUD operations
and an applyPowerUp method. Create Express routes at /api/powerups.
Update the battle system so power-ups can be used during battle.

⏸️ Press Enter and wait. Let the agent work — do not interrupt!

👀 What They See:

Agent starts planning its approach, then begins creating new files, editing existing ones, and potentially running commands — all automatically. Multiple files appear in the editor as it works.

💡 Teaching Point:

One prompt → multiple files created and connected. This is the power of Agent Mode over Edit Mode — it handles the full scope of a feature, not just a single file.

⚠️ Important — Read This Before Continuing

Let the agent work! Resist the urge to intervene. The audience needs to see the full autonomous flow — planning, file creation, code writing, and self-organization. Interrupting breaks the magic of the demo. If it makes a mistake, that's even better — it sets up the self-correction step later.

3

Narrate What's Happening

💬 Say:

"Watch how it plans its approach, creates files in order, and connects everything together."

👉 Point out as it happens — narrate the live activity:

  • 📄 New files being created in the explorer
  • ✍️ Code appearing in real-time in the editor
  • 🔗 Import statements being added to connect modules
  • 💻 Terminal commands running (if the agent triggers any)
👀 What They See:

Real-time file creation in the sidebar, code appearing in the editor tabs as the agent writes it, and potentially terminal commands executing automatically.

💡 Teaching Point:

Agent Mode understands file relationships — it creates interfaces before services, services before routes. It builds a dependency graph and works through it in the right order.

4
4

Review the Output

👉 Walk through created files in the explorer sidebar. Open each one briefly.

💬 Say:

"Let's check — did it follow our coding standards from copilot-instructions.md?"

🔍 Check the code for:

  • ✅ JSDoc comments on functions and interfaces
  • ✅ const declarations (not let or var)
  • ✅ Proper error handling (try/catch, status codes)
  • ✅ Consistent code style with the rest of the project
👀 What They See:

Clean, well-structured code that follows the project's established patterns. The generated code looks like it was written by someone who read the project's style guide.

💡 Teaching Point:

Agent Mode follows the loop: plan → execute → self-correct. It respects your project's rules defined in copilot-instructions.md and adapts to the existing code style automatically.

5
5

Show Self-Correction 🔄

👉 If the agent didn't already run tests, type the follow-up prompt:

⌨️ Type in Agent Chat: 
Now run the tests and fix any failures
💬 Say:

"The iteration loop is the real magic of Agent Mode."

⏸️ Let it run. Watch the cycle: run tests → see failures → read errors → fix code → re-run tests.

👀 What They See:

Agent runs the test suite, sees failures, reads the error messages, navigates to the failing code, fixes it, and re-runs the tests — all without any additional input from you. It may go through multiple fix-test cycles.

💡 Teaching Point:

Self-correction is what separates Agent Mode from simple code generation. It can debug its own work — read error output, understand what went wrong, and fix it. This is the closest thing to pair programming with an AI.
