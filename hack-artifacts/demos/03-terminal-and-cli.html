🚀 Copilot in Terminal + GitHub CLI

Live demo — natural-language shell commands, PR creation, and issue management with Copilot & gh

🎬 Setup — Before You Start

  • VS Code open with an integrated terminal in the ../demo-app/ directory (Emoji Battle Arena project)
  • GitHub CLI installed and authenticated — verify with gh auth status
  • Copilot CLI installed — verify with copilot --version
  • Terminal font size increased for audience visibility (≥ 16 pt recommended)
  • A feature branch ready with a small change for the PR demo (e.g., a one-line fix in battleService.ts)

📟 Section 1 — Copilot in the Terminal

Natural-language shell superpowers ⏱️ 7 min

1
1 copilot — Natural Language → Shell Commands
💬 Say:

"Copilot isn't just in your editor — it's in your terminal too. Watch this: I'll ask a plain-English question and get a working shell command back."

⌨️ Type:
copilot
    how do I find which TypeScript files have TODO comments
👀 What They See:

Copilot suggests a grep/find command like grep -rn 'TODO' --include='*.ts' src/. The terminal shows the suggested command with an option to copy or run it.

💡 Teaching Point:

Natural language → shell commands. You don't need to memorize grep flags or arcane find syntax anymore. Just describe what you want.

2
2 copilot — Decode Cryptic Commands
💬 Say:

"Found a cryptic command in a README or on Stack Overflow? Copilot can explain it piece by piece."

⌨️ Type:
copilot
    Explain this command: find src -name '*.ts' -exec grep -l 'TODO' {} \;
👀 What They See:

Copilot breaks down each part of the command: find src searches the src directory, -name '*.ts' filters to TypeScript files, -exec runs a command on each match, grep -l lists files with matches, and \; terminates the -exec clause.

💡 Teaching Point:

Great for understanding unfamiliar commands from Stack Overflow, documentation, or teammates. It's like having a senior engineer explain things on demand.

3
3 ⚡ Quick Terminal Trick — Framework-Aware Suggestions
💬 Say:

"It knows your tech stack too. Watch — I'll ask about Vitest specifically."

⌨️ Type:
copilot
    how to run only the failing tests in vitest
👀 What They See:

Copilot suggests vitest-specific commands like npx vitest --reporter=verbose --bail or similar flags. It doesn't give generic Jest advice — it knows the difference.

💡 Teaching Point:

Copilot knows your tech stack — framework-specific commands just work. Whether it's Vitest, pytest, Maven, or Cargo, you get the right answer for your tools.

🔗 Section 2 — GitHub CLI Workflows

Ship code and manage work from the terminal ⏱️ 8 min

4
4 🚢 PR Creation — Ship from the Terminal
💬 Say:

"The GitHub CLI is how Copilot connects to your GitHub workflow. Let's create a PR without leaving the terminal."

⌨️ Type:
gh pr create --title "Fix battle damage calculation" --body "Fixed operator precedence bug in battleService.ts"
👀 What They See:

Terminal shows the PR being created with a direct link to the new PR on GitHub. The audience sees the full flow: code → commit → PR in seconds, all from the terminal.

💡 Teaching Point:

Copilot-assisted PR workflow — from finding the bug to shipping the fix. No context-switching to the browser. Everything stays in your flow.

5
5 📋 Issue Management — Create Work Items
💬 Say:

"Issues are the input for our autonomous agents later. Let's create one with a clear description."

⌨️ Type:
gh issue create --title "Implement leaderboard service" --body "The leaderboard service is currently a stub. Implement getLeaderboard() and getTopFighters()."
👀 What They See:

Issue created on GitHub with a direct link. The issue has a clear title and an actionable body — exactly the kind of description that gives an AI agent (or a human!) enough context to start working.

💡 Teaching Point:

Well-written issues become instructions for the Copilot coding agent. Think of issues as your prompts for autonomous development. Quality in → quality out.

6
6 🙋 Quick Audience Try — Hands-On Moment
💬 Say:

"Your turn! Open your terminal and try copilot with any shell question you've been wondering about. Maybe something you always Google."

⏳ Audience Activity:

Give 2 minutes for hands-on exploration. Walk around if possible. Encourage people to share interesting suggestions they get. Call out any great examples you spot.

💡 Teaching Point:

Hands-on moments keep energy up — let people try it themselves. This is where the "aha" moments happen. Real learning requires real doing.
